# Google Play 심사용 앱 접근 정보 (제출 템플릿)

이 문서는 Google Play Console > 앱 콘텐츠(App content) > 앱 액세스(App access) 항목에 입력할 내용을 정리한 템플릿입니다.

## 1) 기본 원칙

- 심사자는 앱의 모든 기능에 접근할 수 있어야 합니다.
- 심사자는 회원가입, 무료 체험 등록, 개발자 문의를 대신 수행하지 않습니다.
- 로그인, 2단계 인증, 위치 제한, 구독/멤버십, 다른 기기 연동이 필요한 경우 개발자가 사전 제공해야 합니다.

- 테스트 계정 ID(이메일/닉네임/전화번호): `[REVIEW_TEST_ID]`
- 테스트 계정 비밀번호: `[REVIEW_TEST_PASSWORD]`
- 계정 상태: `활성 / 정지 아님 / 만료 아님`
- 계정 권한: `일반 사용자 + 심사에 필요한 모든 메뉴 접근 가능`

주의:
- 테스트 계정은 심사 기간 동안 비밀번호/권한/상태를 변경하지 않습니다.
- CAPTCHA, 기기 제한, IP 차단, 지역 차단 등 자동 방어가 심사 계정에 적용되지 않도록 예외 처리합니다.

## 8) 이 저장소에서 심사용 계정 자동 생성

아래 명령은 DB에 심사용 계정을 생성(또는 기존 닉네임이면 비밀번호 갱신)합니다.

```bash
DATABASE_URL='postgresql://USER:PASS@HOST:5432/DB' \
REVIEW_NICKNAME='gp_review_test' \
REVIEW_PASSWORD='StrongPass!234' \
REVIEW_REAL_NAME='Google Play Reviewer' \
npm run create:review-account
```

옵션:
- `REVIEW_PASSWORD`를 생략하면 랜덤 비밀번호가 생성되어 출력됩니다.
- 동일 `REVIEW_NICKNAME`이 이미 존재하면 계정을 새로 만들지 않고 비밀번호/약관 동의 정보만 업데이트합니다.
