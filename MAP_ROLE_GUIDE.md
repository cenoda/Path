# MAP Role Guide (학습형 메타버스 운영 기준)

## 1) 맵 역할 선언문
맵은 이동 공간이 아니라 유저 행동을 만드는 게임 보드다.

- 들어오면: 지금 할 일(오늘 미션)이 보인다.
- 행동하면: 월드가 즉시 바뀐다.
- 끝나면: 성과가 눈에 보이는 형태로 남는다.
- 대학교는: 단순 배경이 아니라 경쟁/소속/목표를 보여주는 핵심 허브다.

## 2) 제품 목표와 맵의 관계

### 핵심 목표
- 재방문율 상승
- 세션 길이 증가
- 친구와 함께 머무는 시간 증가

### 맵이 담당할 일
- 동기 부여: 오늘 무엇을 하면 이득인지 즉시 제시
- 루프 고정: 다음 행동 버튼을 항상 제공
- 성과 시각화: 숫자를 공간 변화로 번역

## 3) 맵 핵심 루프
1. 입장
2. 오늘 미션 확인
3. 친구/파티와 활동 시작
4. 완료 보상 수령
5. 월드 변화 확인
6. 다음 미션 선택

## 4) 존(Zone) 구조

### A. Central Plaza (중앙 광장)
- 역할: 진입 허브 + 오늘 미션 보드
- 핵심 UI: 오늘 미션 3개, 빠른 시작 버튼, 라이브 이벤트 배너
- 성공 기준: 진입 후 10초 내 첫 행동 클릭

### B. Social Arena (소셜 아레나)
- 역할: 친구와 실시간 상호작용
- 핵심 UI: 파티 생성/참가, 근접 채팅, 이모트, 미니게임 입장
- 성공 기준: 동시 접속자 간 상호작용 비율 증가

### C. University District (대학교 디스트릭트)
- 역할: 대학 소속감 + 대학 간 경쟁의 중심
- 핵심 UI: 대학별 허브, 대학 랭킹, 대학 퀘스트, 대학 배너/랜드마크
- 성공 기준: 대학 허브 방문률, 대학 퀘스트 참여율 상승

### D. Identity Home (개인 공간)
- 역할: 자기표현과 장기 보상
- 핵심 UI: 커스터마이즈, 진화 오브젝트, 트로피 월
- 성공 기준: 주간 꾸미기 액션 반복

### E. Event Gate (이벤트 게이트)
- 역할: 시간표 기반 콘텐츠
- 핵심 UI: 남은 시간, 참여 인원, 즉시 참가
- 성공 기준: 이벤트 참여율/재참여율 증가

## 5) 대학교 맵 표현 원칙

### 대학을 어떻게 보이게 할지
- 각 대학은 고유 랜드마크(건물/상징물/색상)로 즉시 식별 가능해야 한다.
- 랜드마크는 장식이 아니라 상호작용 가능한 허브여야 한다.
- 대학별 혼잡도/활동량은 빛, 배너, 파티클 강도로 실시간 표시한다.

### 대학 허브에서 제공할 기능
- 대학 게시판: 오늘 대학 미션, 공지, 주간 목표
- 대학 라운지: 같은 대학 유저 빠른 합류/채팅/파티 생성
- 대학 대전: 대학 간 주간 점수전, 이벤트 매치
- 입시 정보 링크: 대학별 입학처/모집요강 바로가기

### 대학 선택/소속 UX
- 첫 진입 시 대학 선택 또는 검색을 제공한다.
- 소속 대학은 프로필, 이름표, 미니맵에 일관되게 노출한다.
- 대학 변경은 시즌 단위 또는 쿨다운으로 운영해 악용을 방지한다.

## 6) 기능 재정리 (Must / Should / Later)

### Must (지금 바로)
- 중앙 광장 오늘 미션 보드
- 맵 내 즉시 행동 버튼(시작, 참가, 보상)
- 보상 반영 후 월드 즉시 변화(조명, 오브젝트, 이펙트)
- 친구/파티 빠른 입장 동선
- 대학교 허브 노드(랜드마크 + 클릭 진입)
- 대학별 랭킹/활동량 오버레이
- 대학 허브 내 빠른 합류 버튼

### Should (다음 스프린트)
- 대학 대항 점령/공헌 시스템
- 시간표 기반 라이브 이벤트
- 개인 공간 진화 시스템
- 대학 게시판/대학 미션 시스템

### Later (검증 후)
- 시즌 세계관 대규모 업데이트
- 프리미엄 연출 확장
- 대규모 UGC(유저 제작 콘텐츠) 도입
- 대학별 커스텀 테마 시즌

## 7) 지표(KPI)
- 맵 진입 후 첫 행동 클릭률
- 맵 내 평균 체류 시간
- 파티/친구 동반 세션 비율
- 주간 재방문율(D7)
- 이벤트 재참여율
- 대학 허브 방문률
- 대학 미션 완료율
- 대학 대항전 참여율

## 8) 운영 원칙
- 기능 추가 전 질문: 이 기능이 행동 루프를 강화하는가?
- 추가된 기능은 2주 내 KPI 영향 검증
- 영향이 없으면 단순화/제거
- 대학 콘텐츠는 실제 유저 행동(참가/합류/재방문)을 만들지 못하면 축소한다.

## 9) 의사결정 체크리스트
- 맵에 들어오자마자 할 일이 보이는가?
- 행동 직후 월드 변화가 즉시 보이는가?
- 친구와 함께 할 이유가 명확한가?
- 유저가 다음 행동으로 자연스럽게 이어지는가?
- 이 기능이 없으면 지표가 떨어지는가?
- 대학 허브가 단순 장식이 아니라 반복 방문 이유를 만드는가?

## 10) 지금 코드베이스 적용 우선순위
1. 중앙 광장에 오늘 미션 패널 고정 노출
2. 미션 완료 시 월드 오브젝트 상태 변경 훅 추가
3. 파티 입장/합류 버튼을 메인 동선으로 승격
4. 이벤트 게이트 타이머/참여 버튼 UI 추가
5. 개인 공간 진화 상태를 프로필과 연결
6. 대학 랜드마크 클릭 시 대학 허브 패널 오픈
7. 대학별 활동량 기반 랜드마크 이펙트 레벨 적용
8. 대학 미션/랭킹 API를 허브 패널에 연결

## 11) 이번 스프린트 목적 고정 (흔들림 방지)

### 스프린트 단일 목적
- 유저가 맵에 들어온 뒤 10초 안에 첫 행동을 시작하게 만든다.

### 스프린트 성공 정의
- 맵 진입 후 첫 행동 클릭률이 기준 대비 상승한다.
- 파티/합류 버튼 클릭률이 기준 대비 상승한다.
- 기능 추가 없이도 다음 행동 전환률이 유지 또는 개선된다.

### 이번 스프린트 비목적 (하지 않을 것)
- 대규모 신규 존 추가
- 대형 세계관 연출 확장
- 장식성 UI 개편(행동 전환과 무관한 변경)
- 대학 시스템 전면 리뉴얼

## 12) 실행 보드 (Must 8개를 작업 단위로 분해)

### Track A. 첫 행동 10초 달성
- A1: 중앙 광장 오늘 미션 패널 항상 보이기
- A2: 오늘 미션 카드에 즉시 시작 버튼 붙이기
- A3: 맵 진입 직후 포커스/가이드 표시

### Track B. 친구와 즉시 합류
- B1: 파티 입장/합류 버튼을 메인 HUD 상단 고정
- B2: 친구 온라인 상태와 합류 가능 여부 노출
- B3: 합류 실패 사유를 즉시 안내(가득 참/권한 없음)

### Track C. 행동 후 월드 변화 체감
- C1: 미션 완료 이벤트 훅 표준화
- C2: 조명/오브젝트/이펙트 반영 최소 1개 이상 보장
- C3: 변화 확인 후 다음 행동 CTA 자동 제시

### Track D. 대학 허브 반복 방문
- D1: 대학 랜드마크 클릭 -> 허브 패널 오픈
- D2: 대학 활동량 오버레이 표시
- D3: 대학 허브 내 빠른 합류 버튼 제공

## 13) 주간 의사결정 룰

### 기능 추가 승인 조건
- 아래 3개 중 2개 이상을 만족할 때만 개발 착수
- 첫 행동 클릭률 개선에 직접 기여한다.
- 파티/친구 동반 세션 비율 개선에 기여한다.
- 대학 허브 방문/참여 지표 개선에 기여한다.

### 기능 중단 조건
- 배포 후 2주 내 KPI 변화가 없거나 악화
- 유지보수 비용이 행동 루프 기여도보다 큼
- 유저가 다음 행동으로 이어지지 않음

### 회의 운영 규칙
- 모든 제안은 "어떤 KPI를 얼마나 올릴지" 숫자로 제출
- 숫자 없는 제안은 백로그로만 이동(당장 개발 금지)
- 회의 마지막에 이번 주 단일 목적 1줄 재확인
